# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有橙白柯基 IP。
- 柯基没有尾巴。
- 柯基承担核心动作，不只是装饰。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面萌、亲和、有意思，像正文插画里的个人 IP 配图。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 柯基像 3D 吉祥物、宠物头像、纯表情包或低幼儿童卡通。
- 只有角色卖萌，没有帮读者理解文章内容。
- 柯基出现尾巴、短尾、卷尾或尾巴轮廓。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 和旧案例或外部参考图的构图过于相似。

## 迭代方法

- 太普通：让柯基成为动作主体，加入一个轻松、好懂、有 IP 感的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太幼稚：强调 cute but article-focused, warm but not childish, IP-like but not pure sticker。
- 太像贴纸：强调 in-article illustration, integrated with the idea, not standalone sticker。
- 太 3D：强调 hand-drawn line art, flat white background, no 3D render, no plush texture。
- 有尾巴：局部编辑去掉尾巴；重生成时明确 no tail, no short tail, no curled tail。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和柯基动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“这是正文里的一张柯基 IP 插画，它在认真帮我理解这件事”，然后 1 秒内看懂结构。

如果第一眼像教程页、流程图，或只是可爱贴纸而没有内容表达，就不合格。
