# 风格 DNA

## 一句话

纯白、清爽、正文插画、手绘、留白、萌系 IP、亲和、轻松、中文手写感、结构清楚但不说明书。

像一个有个人 IP 的内容创作者，在文章正文里放的一张轻松解释图。柯基 IP 是动作主体，可以可爱、圆润、有回应感，但画面首先要像正文插画，不能变成纯宠物海报或贴纸表情包。

## 必须

- 16:9 横版正文配图。
- 纯白背景：不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- 黑色手绘线稿为主：细线、轻微抖动、不机械、不矢量、不厚重描边。
- 柯基使用少量橙白识别色：橙色耳朵和身体、白色脸部中线和胸口、黑鼻子。
- 角色可以更圆润、更萌、更接近参考图的亲和感，但整体仍保持正文手绘插画，不要 3D 渲染。
- 柯基没有尾巴：任何角度都不要画尾巴、短尾、卷尾或尾巴轮廓。
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，最好有一整块安静区域。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 结构要自然表达，不要在图上写结构类型名称。

## 颜色

- 黑色：主体线稿、角色、框线、结构、主要文字、主体物件。
- 红色：重点批注、问题、情绪点、关键提醒、结果。
- 橙色：柯基识别色、主流程、路径、箭头、自动化流向、从 A 到 B 的移动关系。
- 蓝色：补充说明、脑内状态、系统状态、第二层解释、AI/assistant/自动化提示。

蓝色不是每张都必须用。颜色要克制，宁可少不要多。

## 绝对不要

- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要低幼儿童插画。
- 不要 3D 吉祥物渲染。
- 不要宠物写真、纯贴纸表情包、毛绒玩具感。
- 不要只有角色卖萌、没有表达文章结构。
- 不要给柯基加尾巴。
- 不要复杂架构图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理。
- 不要把每个节点都解释清楚。
- 不要左上角写“Workflow 流程图 / 系统架构图 / 常见坑 / 路线图”等类型标题。

## 审美方向

要像正文插画，同时保留萌、亲和、轻松、有意思、简洁清爽和一点小巧思。

可以可爱，但不要低幼；可以像 IP 形象，但不要复杂、死板或只剩装饰。
