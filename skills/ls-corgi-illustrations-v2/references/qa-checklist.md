# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底或极浅米白,没有纸纹、噪点、复杂图案。允许极淡阴影做主体投影。
- 有 LS-Q。
- LS-Q 外形对得上:Q 版大头比例（头身 1:1.2~1:1.5）、暖橙主体、米白面部 V 形白纹、三角立耳（粉内耳）、大圆眼、黑小圆鼻、微笑。
- LS-Q 承担核心动作,不只是装饰或站着看。
- 画风是扁平化 3D 卡通:色块干净 + 立体光影 + 明暗渐变。
- 没有复刻旧案例或 reference-ip 里的场景,而是为当前文章生成了新隐喻。
- 画面可爱、活泼、温暖。
- 简洁,主体不超过画面约 55%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头,不与 LS-Q 本体暖橙混淆。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况,重生成或局部编辑:

- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- LS-Q 外形漂移:颜色变棕或灰、耳朵耷拉、面部白纹消失、腿变长、比例从 Q 版大头变正常柯基比例、表情冷酷。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、强噪点、米色、复杂图案。
- 真实 UI 截图或科技感界面。
- 画风是纯 2D 平面（没有立体感）或超写实 3D 渲染（毛发颗粒、照片材质）。
- 中文错字严重或标注不可读。
- 画面太冷淡,没有可爱感。
- LS-Q 没在做活,只是微笑站着。
- LS-Q 被服装道具淹没（穿太多件、道具堆叠）。

## 迭代方法

- 太普通:让 LS-Q 成为动作主体,加入一个可爱但成立的隐喻。
- 太复杂:删节点,只保留一个动作和 3-5 个短标注。
- 太冷淡:强调 brighter eyes, warmer smile, cute mascot feeling, soft 3D volumetric lighting。
- 太 PPT:去掉标题、边框、整齐网格和过多箭头,改成手绘场景。
- 太像旧案例或 reference-ip:保留核心意思,换掉主物件和 LS-Q 动作。
- LS-Q 漂移:用 prompt-template 里的"修正 LS-Q 外形漂移"提示词重生。
- 文字错:优先局部编辑;错得多就重生成并减少标注数量。
- 服装道具过多:减到 1-2 件服装 + 1 个道具。

## 交付判断

高质量图应该让读者先觉得"可爱、有意思",然后 1 秒内看懂结构,同时被 LS-Q 的卖萌认真感染一下。

如果第一眼像教程页、PPT 流程图或冷酷商业插画,而不是白底上的 Q 版 3D 卡通可爱插画,就不合格。
