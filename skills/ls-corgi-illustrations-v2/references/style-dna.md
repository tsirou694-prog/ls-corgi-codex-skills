# 风格 DNA

## 一句话

纯白底、Q 版大头比例、扁平化 3D 卡通、立体光影、可爱卖萌、中文手写批注、结构清楚但不说明书。

像一只可爱活泼的暖橙色柯基小助手,正在帮你把抽象概念演成一张可读又讨喜的 Q 版 3D 卡通插画。

## 必须

- 16:9 横版正文配图。
- 极简背景:首选纯白,次选极浅米白或极浅暖灰。不要米色纸纹、不要噪点、不要复杂背景图案。允许极淡的渐变阴影做主体投影。
- 主体:LS-Q 柯基 Q 版 3D 卡通 + 必要的物件（也用 3D 卡通质感）。
- 色块干净 + 立体光影:明暗渐变、柔和体积感、清晰边缘。介于 2D 平面插画和 3D 渲染之间。
- 大量留白:主体占画面约 40%-55%,至少 35% 空白,最好有一整块安静区域。
- 少量中文手写批注:最多 5-8 处,每处尽量 2-8 个字。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 结构要自然表达,不要在图上写结构类型名称。

## 颜色

- 暖橙色:LS-Q 柯基本体（焦糖橘黄、姜黄）。
- 米白色:LS-Q 面部条纹、腹部、爪尖、嘴部周围。
- 白色:背景。
- 黑色:物件线稿、结构、主要文字、关键描边、机器、路径骨架。
- 红色:重点批注、问题、情绪点、关键提醒、结果。
- 橙色:主流程、路径、箭头、自动化流向、从 A 到 B 的移动关系。
- 蓝色:补充说明、脑内状态、系统状态、第二层解释、AI/assistant/自动化提示。

蓝色不是每张都必须用。颜色要克制,宁可少不要多。LS-Q 本体的暖橙色不要和"橙色箭头/路径"混淆——主体色和功能色要分开。

## 绝对不要

- 不要商业精致插画、不要照片级写实。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要复杂架构图。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、强噪点、复杂纹理。
- 不要把每个节点都解释清楚。
- 不要左上角写"Workflow 流程图 / 系统架构图 / 常见坑 / 路线图"等类型标题。

## 允许（v2 相比 v1 解锁的）

- ✓ Q 版大头比例（头身 1:1.2 ~ 1:1.5）
- ✓ 立体光影、明暗渐变、柔和体积感
- ✓ 可爱卖萌表情（露小白牙、小粉舌、闪亮大眼）
- ✓ 服装（魔法袍、围裙、马甲等，1-2 件）
- ✓ 道具（咖啡杯、魔法棒、放大镜、键盘等）

## 审美方向

要可爱、活泼、温暖、有意思、简洁。

不要冷酷、不要阴郁、不要复杂、不要死板。

LS-Q 的卖萌感是核心魅力来源之一,但要服务结构表达,不要喧宾夺主。
