# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底或极浅米白,没有纸纹、噪点、渐变。
- 有 LS。
- LS 外形对得上:橙黄色身体、白面部条纹、立耳、短腿、黑鼻子、微笑表情。
- LS 承担核心动作,不只是装饰或站着看。
- 没有复刻旧案例或 reference-ip 里的场景,而是为当前文章生成了新隐喻。
- 画面清爽、轻怪诞、有意思。
- 简洁,主体不超过画面约 55%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头,不与 LS 本体橙黄混淆。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况,重生成或局部编辑:

- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- LS 像吉祥物、表情包、卖萌卡通海报、儿童绘本。
- LS 外形漂移:颜色变棕或灰、耳朵耷拉、面部白纹消失、腿变长、表情夸张。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点、复杂图案。
- 真实 UI 截图或科技感界面。
- 3D 渲染、精致矢量、商业插画质感。
- 中文错字严重或标注不可读。
- 画面太死板,没有轻怪诞的隐喻。
- LS 没在做活,只是微笑站着。

## 迭代方法

- 太普通:让 LS 成为动作主体,加入一个奇怪但成立的隐喻。
- 太复杂:删节点,只保留一个动作和 3-5 个短标注。
- 太卖萌:强调 earnest, focused on work, slight smile only, not cute mascot, not children's book。
- 太 PPT:去掉标题、边框、整齐网格和过多箭头,改成手绘场景。
- 太像旧案例或 reference-ip:保留核心意思,换掉主物件和 LS 动作。
- LS 漂移:用 prompt-template 里的"修正 LS 外形漂移"提示词重生。
- 文字错:优先局部编辑;错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得"有点意思/有点怪",然后 1 秒内看懂结构,同时被 LS 的笨拙认真感染一下。

如果第一眼像教程页、卖萌海报或商业插画,而不是白纸上的轻怪诞产品草图,就不合格。
